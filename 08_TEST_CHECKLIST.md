# Тест-чеклист

## База
- [ ] База создаётся в `/data/bot.db`.
- [ ] После Redeploy данные сохраняются.
- [ ] Дубликат активного Threads username не создаётся.
- [ ] Архивирование не удаляет историю.
- [ ] Повторная invite-привязка не вызывает UNIQUE-ошибку.

## Админ
- [ ] `/start` показывает админ-меню.
- [ ] Клиент создаётся после подтверждения.
- [ ] Telegram username принимается в нескольких форматах.
- [ ] Тема создаётся автоматически.
- [ ] Карточка показывает корректные статусы.
- [ ] Таблица подключается.
- [ ] Контент-план подключается.
- [ ] Клиент архивируется.

## Клиент
- [ ] Invite-ссылка подключает кабинет.
- [ ] Клиент видит только свои кнопки.
- [ ] Ветки выдаются отдельными сообщениями.
- [ ] Черновики не выдаются.
- [ ] Подтверждение публикации сохраняется.
- [ ] Опрос раз в 2 дня сохраняется.
- [ ] Сообщение менеджеру отправляется.

## Рабочая тема
- [ ] Создаётся для правильного клиента.
- [ ] Действия клиента дублируются.
- [ ] Сообщение клиента приходит в тему.
- [ ] Ответ из темы приходит клиенту.
- [ ] Клиент не видит тему.

## Аналитика
- [ ] Внесение пятничной статистики работает.
- [ ] Данные привязаны к правильной неделе.
- [ ] Процент публикации считается корректно.
- [ ] Сравнение периодов корректно.
- [ ] Архивные клиенты не входят в активную сводку.
