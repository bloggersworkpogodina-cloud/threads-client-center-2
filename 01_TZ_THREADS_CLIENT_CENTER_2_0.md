# THREADS CLIENT CENTER 2.0
## Техническое задание

### Цель
Создать стабильного Telegram-бота для управления клиентами по Threads из одного админ-центра.

Бот является единым рабочим полем:
- клиент получает ветки;
- подтверждает публикацию;
- сообщает результаты;
- получает контент-план;
- общается с менеджером;
- администратор управляет клиентами и видит аналитику.

Рабочий Telegram-чат с темами является вспомогательным каналом:
- общение менеджера с клиентом;
- дублирование ключевых действий клиента;
- просмотр истории действий клиента в боте;
- внутренний контроль.

### Роли

#### Администратор
Видит:
- всех клиентов;
- карточку каждого клиента;
- подключение Google-таблицы;
- подключение контент-плана;
- историю отправленных веток;
- подтверждения публикации;
- ответы по откликам и заявкам;
- пятничную статистику;
- аналитику;
- внутренние события;
- архив.

#### Клиент
Видит только:
- свои ветки;
- свой контент-план;
- вопросы по публикации;
- вопросы по откликам и заявкам;
- связь с менеджером;
- ответы менеджера.

Клиент не видит:
- других клиентов;
- общую аналитику;
- внутренние комментарии;
- рабочую тему;
- служебные события;
- административные функции.

#### Менеджер
Работает в персональной теме клиента:
- получает сообщения клиента;
- отвечает клиенту;
- видит дубли ключевых действий клиента;
- видит служебную историю по клиенту.

### Основной функционал

#### Создание клиента
Администратор вводит:
1. Имя клиента.
2. Threads username.
3. Telegram username.
4. Подтверждает карточку.

Бот:
- нормализует Threads username;
- принимает Telegram username в форматах `username`, `@username`, `https://t.me/username`, `-`;
- не создаёт дубль активного клиента;
- создаёт персональную тему клиента в рабочем чате;
- сохраняет `topic_id`;
- создаёт invite-ссылку;
- показывает карточку клиента.

#### Клиентский кабинет
Кнопки:
- `📅 Ветки`
- `📄 Контент-план`
- `📊 Результаты`
- `💬 Связь с менеджером`

#### Ветки
Бот ежедневно:
- читает отдельную Google-таблицу клиента;
- выбирает строки на текущую дату;
- выбирает только статус `Готово`;
- отправляет каждую ветку отдельным сообщением;
- дублирует факт отправки в рабочую тему клиента.

В конце дня бот спрашивает:
- все ветки опубликованы;
- опубликована часть;
- не опубликованы.

Ответ сохраняется в базе и дублируется в тему клиента.

#### Результаты раз в 2 дня
Бот спрашивает:
- сколько было откликов;
- сколько было заявок;
- дополнительный комментарий при необходимости.

Ответ:
- сохраняется в базе;
- привязывается к клиенту и периоду;
- дублируется в персональную тему.

#### Пятничная статистика
Администратор раз в неделю открывает карточку клиента и вводит статистику аккаунта.

Минимальный набор:
- просмотры;
- лайки;
- ответы;
- репосты;
- цитирования;
- новые подписчики;
- переходы в Telegram;
- лучшая ветка;
- комментарий менеджера.

Данные сохраняются по неделям.

#### Аналитика
Администратор видит по каждому клиенту:
- сколько веток было отправлено;
- сколько опубликовано;
- дисциплину публикаций;
- количество откликов;
- количество заявок;
- недельную статистику;
- динамику по периодам;
- эффективность контента;
- эффективность публикационной дисциплины.

#### Связь с менеджером
Клиент пишет через кнопку `Связь с менеджером`.

Бот:
- отправляет сообщение в персональную тему клиента;
- менеджер отвечает в теме;
- ответ возвращается клиенту в личный кабинет.

### Архивирование
Закрытие проекта:
- переводит клиента в архив;
- не удаляет историю;
- убирает клиента из активного списка;
- сохраняет все связи, отчёты и события.

### Что не входит в первый релиз
- автопубликация в Threads;
- AI-генерация веток;
- Threads API;
- массовые рассылки;
- отдельная веб-CRM;
- автоматическое изменение статусов Google Sheets.
