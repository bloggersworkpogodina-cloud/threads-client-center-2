# THREADS CLIENT CENTER 2.0

Чистый MVP Telegram-бота для управления клиентами Threads.

## Структура

- `bot.py` — интерфейс администратора и клиента, invite-ссылки, темы менеджеров.
- `db.py` — SQLite, миграции и все операции с клиентами.
- `google_sheets.py` — чтение первого листа Google-таблицы по заголовкам.
- `migrations/001_initial.sql` — первая схема БД и уникальные индексы.
- `smoke_test.py` — локальная проверка базы и критичных сценариев.
- `railway.json` — команда запуска Railway.

## Railway

1. Создайте новый отдельный проект/сервис. Старый не удаляйте.
2. Добавьте Volume и укажите mount path `/data`.
3. Добавьте переменные из `.env.example`.
4. Обязательно установите `DB_PATH=/data/bot.db`.
5. Загрузите проект в GitHub или Railway и выполните deploy.

## Google Sheets

1. Создайте сервисный аккаунт Google Cloud и включите Google Sheets API и Google Drive API.
2. Скопируйте весь JSON-ключ в Railway-переменную `GOOGLE_SERVICE_ACCOUNT_JSON`.
3. Расшарьте каждую клиентскую таблицу на email сервисного аккаунта как читателю или редактору.
4. Первый лист должен иметь заголовки: `Дата`, `Время`, `Ветка`, `Статус`.
5. Даты: `ДД.ММ.ГГГГ` или `YYYY-MM-DD`; статус для выдачи: `Готово`.

## Рабочая Telegram-группа

- Группа должна быть супергруппой с включёнными темами.
- Бот должен быть администратором и иметь право создавать темы и отправлять сообщения.
- В `WORK_GROUP_ID` укажите отрицательный ID группы, обычно начинающийся с `-100`.

## Локальная проверка

```bash
python -m py_compile bot.py db.py google_sheets.py smoke_test.py
python smoke_test.py
```

## Обязательный тест-чеклист

- [ ] `/start` администратора показывает админ-меню.
- [ ] Новый клиент создаётся один раз и получает invite-ссылку.
- [ ] Повторный Threads username не создаёт дубль.
- [ ] Invite-ссылка подключает клиента; повторное открытие не вызывает UNIQUE-ошибку.
- [ ] Карточка показывает статус «подключён».
- [ ] Ссылка Google Sheets сохраняется в карточке клиента.
- [ ] «Ветки на сегодня» выдаёт отдельными сообщениями только строки за сегодня со статусом «Готово».
- [ ] Контент-план открывается по персональной ссылке.
- [ ] Сообщение клиента попадает в его тему.
- [ ] Ответ менеджера из темы возвращается клиенту.
- [ ] Закрытый проект исчезает из активного списка, но остаётся в БД.
- [ ] После Redeploy сохраняются клиент, Telegram-привязка, тема и ссылки.

## Безопасное переключение

Сначала тестируйте новый сервис на отдельном тестовом токене. Рабочий токен переносите только после полного чек-листа и тестового Redeploy. Старый сервис не удаляйте минимум 7 дней.
